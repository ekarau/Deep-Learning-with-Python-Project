## Batch Normalization

### Internal Covariate Shift

The core problem BN solves: as Layer 1 updates its weights, it changes the distribution of data sent to Layer 2. Every layer is constantly learning a **moving target**. BN stabilizes this.

### Four Steps Per Mini-Batch

1. Compute batch mean: $\mu_B = \frac{1}{m}\sum x_i$
2. Compute batch variance: $\sigma_B^2 = \frac{1}{m}\sum(x_i - \mu_B)^2$
3. Normalize: $\hat{x}_i = (x_i - \mu_B) / \sqrt{\sigma_B^2 + \varepsilon}$
4. Scale and shift: $y_i = \gamma \hat{x}_i + \beta$

### Why Learnable γ and β?

Raw normalization (steps 1–3) forces mean=0, var=1. This is too restrictive — Sigmoid is approximately linear near zero, so forcing inputs there kills the non-linearity. The learnable $\gamma$ (scale) and $\beta$ (shift) let the network find the optimal distribution. If the network learns $\gamma = \sigma_B$ and $\beta = \mu_B$, normalization is perfectly undone — BN can be a no-op if that's best. The network is **never worse off** for having BN.

### Layer Norm vs Batch Norm

- **Batch Norm:** Statistics across the **batch dimension**. Breaks with batch_size=1, struggles with variable-length sequences.
- **Layer Norm:** Statistics across **features of a single sample**. Independent of batch → standard for Transformers (BERT, GPT).

### Phase 2 Experiments

| Experiment | Result | Insight |
|---|---|---|
| ReLU + BN | ~98% (vs ~97% without) | Marginal improvement — ReLU already works well |
| Sigmoid + BN | ~97% (vs ~11% without!) | **Fully rescued.** BN normalizes pre-activations into non-saturating region |

The Sigmoid+BN result is the most important finding of Phase 2: it **confirms that Sigmoid's failure was entirely due to pre-activation saturation**, not any inherent weakness of the function itself. When pre-activations are kept in the linear region, Sigmoid performs comparably to ReLU.

## Bonus: LR Scheduler (StepLR)

### Motivation

A constant learning rate forces a compromise: too high → oscillates in late training, too low → wastes early epochs. A scheduler dynamically adjusts $\varepsilon$:

$$\varepsilon_t = \varepsilon_0 \cdot \gamma^{\lfloor t / \text{step\_size} \rfloor}$$

I configured `step_size=3, gamma=0.5`: every 3 epochs, LR halves. With 15 epochs this gives a clear staircase pattern.

## Error Analysis: Confusion Matrix

### Implementation

```python
def get_preds(model, loader):
    model.eval()
    preds, labels = [], []
    with torch.no_grad():
        for x, y in loader:
            preds.extend(model(x.to(device)).argmax(1).cpu().numpy())
            labels.extend(y.numpy())
    return np.array(labels), np.array(preds)
```

## Conclusions & Recommendations

I synthesized all team members' findings into the final conclusions section:

### Key Findings Summary

**Week 2:** Bias-Variance tradeoff demonstrated. MLE + Gaussian = MSE. MAP + priors = regularization.

**Phase 1 (Core):** Sigmoid fails at ~11% due to $0.25^4 \approx 0.004$ gradient decay. ReLU/LeakyReLU reach ~97%. LeakyReLU has 0% dead neurons (strictly superior).

**Week 4:** L1 drives weights to exact zero (sparsity, Laplace prior). L2 shrinks proportionally (Gaussian prior). Dropout trains $2^n$ ensemble. Label smoothing prevents overconfidence.

**Week 5:** He init compensates ReLU's 50% signal loss. Adam converges fastest. BatchNorm fully rescues Sigmoid — root cause is saturation, not the function itself.

### Design Recommendations Table

| Use Case | Choice | Reason |
|---|---|---|
| Default activation | ReLU | $f'(x)=1$ for $x>0$, no saturation |
| Dead neuron risk | LeakyReLU | $f'(x)=0.01$ for $x<0$, always learns |
| Binary output | Sigmoid + BCE | Output ∈ (0,1) |
| Multi-class output | Softmax + CCE | `CrossEntropyLoss` handles internally |
| Init with ReLU | He (Kaiming) | $\text{Var}=2/n_{in}$ compensates 50% loss |
| Default optimizer | Adam | Momentum + RMSProp + bias correction |
| Regularization | L2 + Dropout | Complementary mechanisms |
| Deep networks | + BatchNorm | Normalizes pre-activations, stabilizes training |
