## L2 Regularization (Weight Decay)

### Theory

L2 adds a penalty proportional to squared weight magnitude:

$$\tilde{J}(\theta) = J(\theta) + \frac{\alpha}{2}\|w\|_2^2$$

The weight update becomes:

$$w \leftarrow (1 - \varepsilon\alpha)w - \varepsilon \nabla_w J$$

The factor $(1-\varepsilon\alpha) < 1$ **multiplicatively shrinks weights every step** (rubber band effect). Geometrically, L2 selectively shrinks weight components along directions with small Hessian eigenvalues — directions the data doesn't strongly need.

**Bayesian interpretation:** L2 = MAP estimation with Gaussian prior $P(w_i) \propto \exp(-w_i^2 / 2\sigma^2)$.

### Implementation

Used PyTorch's built-in `weight_decay` parameter: `optim.SGD(params, lr=0.01, weight_decay=1e-4)`.

## L1 Regularization (Lasso)

### Theory

$$\Omega(\theta) = \|w\|_1 = \sum_i |w_i|, \quad \nabla = \alpha \cdot \text{sign}(w) + \nabla_w J$$

**Key difference from L2:** L2 adds force proportional to $w$ (weakens as $w \to 0$, like a rubber band). L1 adds **constant force** $\pm\alpha$ regardless of weight magnitude → drives weights all the way to **exactly zero** (sparsity). Each feature has threshold $\alpha/H_{ii}$; below it, the weight is eliminated completely.

**Bayesian interpretation:** L1 = Laplace prior $P(w_i) \propto \exp(-|w_i|/b)$. The Laplace distribution has a sharp spike at zero → concentrates probability mass at zero → sparsity.

### Implementation

L1 is NOT a built-in optimizer parameter. I implemented it manually, applied **only to weights** (not biases):

```python
if l1_lambda > 0:
    l1_penalty = sum(p.abs().sum() for name, p in model.named_parameters() if 'weight' in name)
    loss += l1_lambda * l1_penalty
```

Biases are excluded because they don't contribute to overfitting the same way weights do.

## Norm Penalties as Constrained Optimization

The penalty approach (minimize $J + \alpha\Omega$) is equivalent to the constraint approach (minimize $J$ s.t. $\Omega(\theta) \leq k$):

- **L2 constraint region:** Ball (circle in 2D) — smooth surface, contact point typically nonzero everywhere
- **L1 constraint region:** Diamond (rotated square) — sharp corners on axes where one coordinate is exactly zero → loss contours are more likely to touch a corner → sparse solution

## Dropout

### Theory

Each training step generates a binary mask: $\tilde{h}^{(l)} = \mu^{(l)} \odot h^{(l)}$, where $\mu \sim \text{Bernoulli}(p)$.

With $n$ neurons: $2^n$ possible sub-networks → Dropout trains an **exponential ensemble** sharing weights.

**Inverted Dropout (PyTorch default):** Surviving activations divided by $p$ during training so $E[\text{output}] = h$. No scaling needed at inference. Note: `nn.Dropout(p)` parameter is the **drop** probability, not keep.

### Experiment

Compared `use_dropout=True` (p=0.2, our default) vs `use_dropout=False`. On MNIST the difference is subtle because the model is not severely overfitting, but the methodology is correctly applied.

## Label Smoothing

### Theory

Replaces hard one-hot targets with soft targets:
- Correct class: $1 - \varepsilon$
- Each wrong class: $\varepsilon / (k-1)$

For MNIST ($k=10$, $\varepsilon=0.1$): correct class becomes 0.9, each wrong class gets ~0.011.

This prevents the model from pursuing hard probabilities (0 or 1), which would require infinitely large weights through softmax and can never be achieved. It acts as output noise regularization.

### Implementation

```python
loss_fn = nn.CrossEntropyLoss(label_smoothing=0.1)
```

### Experiment

Tested $\varepsilon = 0.1$ (default from Inception-v2, Szegedy et al. 2016). Training loss is higher (expected — targets are softer), but test accuracy is comparable. On clean MNIST labels the effect is subtle.

## Semi-Supervised & Self-Supervised Learning

**Semi-supervised** uses both labeled $(x,y)$ and unlabeled $(x)$ data. Core assumption: features useful for modeling $P(x)$ transfer to predicting $y$.

**Self-supervised** creates artificial tasks from data itself: BERT masks words and predicts them, GPT predicts the next token. No human labels needed. Modern pipeline: (1) pre-train on massive unlabeled data to learn $P(x)$, (2) fine-tune on small labeled data for $P(y|x)$.

---

## Adversarial Training (FGSM)

Fast Gradient Sign Method: $\tilde{x} = x + \varepsilon \cdot \text{sign}(\nabla_x J(\theta, x, y))$

Each input pixel is nudged in the direction that maximally increases the loss. In high-dimensional spaces (~784 pixels), all perturbations push constructively in the same wrong direction. Training on adversarial examples forces smoother decision boundaries → acts as regularization.
